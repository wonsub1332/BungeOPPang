# 스프라이트 최적화 기록 (Sprite Optimization Log)

이 문서는 이미지 스프라이트의 투명 영역 문제 해결 과정과 적용된 기술적 조치를 기록합니다.

## 1. 발생한 문제 (Issues)
* **불필요한 투명 영역 차지:** 이미지 파일의 실제 크기에 비해 투명한 배경이 너무 넓어, 유니티 에디터에서 오브젝트를 배치하거나 클릭할 때 판정이 정확하지 않았음.
* **유니티 설정 충돌:** `Sprite Mode`가 `Multiple`로 설정되어 있어, 이미지가 하나임에도 불구하고 유니티가 전체 영역을 하나의 시트로 인식하고 있었음.
* **미세 픽셀(Noise) 잔류:** 배경을 지우는 과정에서 눈에 보이지 않는 아주 흐린 픽셀들이 외곽에 남아있어, 일반적인 자르기(Trim) 기능으로는 여백이 줄어들지 않았음.

## 2. 해결 과정 (Actions Taken)

### Phase 1: 유니티 설정 변경
* 모든 대상 이미지(`.png.meta`)의 `Sprite Mode`를 `Multiple (2)`에서 `Single (1)`로 변경하여 유니티가 단일 스프라이트로 인식하도록 수정함.

### Phase 2: 이미지 데이터 분석 및 정제
* **Python(Pillow)** 라이브러리를 사용하여 이미지의 픽셀 데이터를 직접 분석함.
* **Aggressive Trim(강제 정제):** 투명도(Alpha)가 4% 이하인 미세한 먼지 픽셀들을 강제로 제거하여 실제 형체만 남김.

### Phase 3: 이미지 크기 재조정 및 메타데이터 업데이트
* 정제된 데이터를 바탕으로 이미지의 최소 경계선(Bounding Box)을 계산하여 이미지를 잘라냄(Crop).
* 유니티가 변경된 이미지 크기를 인식할 수 있도록 `.meta` 파일의 `width`와 `height` 값을 새 크기에 맞춰 강제 업데이트함.

## 4. 기대 효과
* **클릭 판정 개선:** 실제 이미지 근처에서만 클릭이 작동함.
* **리소스 효율화:** 불필요한 투명 픽셀 데이터를 제거하여 메모리 및 성능 효율 향상.
* **배치 편의성:** 인스펙터 창에서 이미지 크기가 실제와 일치하여 씬 구성이 용이해짐.
